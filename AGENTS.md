# Codex 作業ガイド

## 目的
- このリポジトリは、幹事を決めるためのフロントエンドアプリを開発するためのものです。
- 現在はルーレット UI を中心に構成されていますが、将来的には開催履歴の記録と参照にも拡張する前提です。

## 基本方針
- まずはフロントエンドの体験を壊さずに改善することを優先します。
- UI の変更では、PC とスマホの両方で最低限操作できる状態を保ってください。
- 将来の履歴機能を見据えて、データ構造は拡張しやすさを意識してください。
- 不要なライブラリ追加は避け、既存の技術構成に沿って進めてください。
- TypeScript / JavaScript では、特別な理由がない限りアロー関数を優先してください。

## 技術構成
- `Vite`
- `React`
- `TypeScript`
- `Tailwind CSS`

## 開発コマンド
- `npm install`
- `npm run dev`
- `npm run build`
- `npm run preview`

## 実装上の前提
- 画面 UI は `src/` 配下に実装します。
- 候補メンバーの初期データは `src/data/` 配下で管理します。
- ドメイン型は `src/types/` 配下で管理します。
- README には、セットアップ手順と現状の機能概要が分かる状態を保ってください。

## データモデル
- `Member`: 幹事候補メンバー
- `EventRecord`: 開催日、場所、その回の幹事を表す記録

## 変更時の確認
- 変更後は可能な範囲で `npm run build` を実行して、ビルドが通ることを確認してください。
- 仕様が未確定な場合でも、将来の拡張を妨げる固定的な設計は避けてください。
